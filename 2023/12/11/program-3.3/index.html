规划系统，组织系统，高效团队特征

规划系统

功能

  • 规划系统提供项目计划功能
  • 设定项目工作目标
    • 使得相关者共享相同的目标
  • 制订实现目标的方案
    • 取得一致的、获得承诺的工作安排

从需要到目标

相关方

  • 谁的需要（需求）？
    • 相关方：影响项目、受到或自认为受到项目影响的个人、群体或组织

相关方分析

  • 相关者分析的难处
    • 分析的边界：“影响”在何处停止？
    • 分析的粒度：对相关方抽象的程度

PI矩阵简介

  • PI矩阵—权力利益矩阵
    • Power：影响项目决策、活动、结果的权力大小
    • Interest：受到项目决策、活动、结果影响的大小

重要的总是少数

特别的，相关者之间的关系可以看成是一种交换，在交换中产生价值

PI矩阵—权力利益矩阵

PI矩阵—权力利益矩阵，是一种用来分析项目相关方的工具。在这个矩阵中，权力（Power）指的是相关方影响项目决策、活动、结果的能力，而利益（Interest）则是指相关方受到项目决策、活动、结果影响的程度。通过这个矩阵，我们可以更好地理解和平衡各个相关方的权力和利益，从而更有效地推进项目的实施。

  • 右上象限标签为"密切管理"，有一个橙色的愤怒表情符号。

Kano模型

从需求到目标

总体需求的子集

  • 多个相关方，每个相关方都有不同的需求，不可能全部满足，能满足的只是总体需求的一个子集。
  • 应准确划分出这个子集，并把它表示为明确的目标。
    • 需求存在于相关方的思维和心中，而目标由项目管理者明确
    • 项目管理者通过实现目标来满足相关方需求
    • 目标是动态的
    • 目标是项目的一个属性

目标设定的要求：代表性，完备，人类可解，一致，可达成的

  • 有代表性的：目标达成即可满足相关需求
  • 完备的：所有目标达成即可满足所有需要满足的需求
  • 人类可解的：可被人理解的、清晰和简洁的，有助于寻找解决方案的，目标应采用与特定解决方案无关的陈述
  • 一致的：经平衡后应无冲突
  • 可达成的：能够以可用的资源实现的

目标排序

  • 是重要又困难的活动
  • 与需要的分类对应，目标可分为

    • 关键目标（3~7）

      关键目标（3~7）：这些目标是项目必须达成的，且对项目的成功至关重要。通常，这些目标数量有限，通常在3至7个之间。

    • 重要目标

      重要目标：这些目标对项目也很重要，但不像关键目标那样，如果没能实现，项目仍然可以成功。它们通常用来优化项目过程和结果。

    • 理想目标

      理想目标：这些目标是在资源和时间允许的情况下，项目可以追求的目标。实现这些目标可能会使项目的结果更好，但即使没能实现，也不会对项目的成功造成太大影响。

从目标到方案（途径）

要求：“好”，创新，工作集

  • 选择“好”的方案
    • 技术：行还是不行？
    • 商务：值还是不值
  • 创新是核心要素
    • 无结构创新
    • 结构化创新
  • 明确工作范围（工作集）
    • 工作分解结构（Work Breakdown Structure, WBS）
    • 工作说明（Statement of Work, SOW）
    • 工作词典

项目规划的工具

  • 预测
  • 资源配置
    • 预算
    • 排程
  • ......

最优计划和应变

  • 存在一个最优计划，需要尽可能去获取真实、充足的信息，以求获得尽量最优的计划，然后严格按计划实施项目
  • 不存在最优计划，未来是不可预料的，“计划不如变化快”，因此有个大致安排，关注主要里程碑就可以了，关键是要随机应变。

组织系统

组织功能和关系

  • 组织系统提供组织功能
  • 设置项目组织结构
  • 组建团队
  • 为成员分配项目工作/任务
  • 组织功能将项目工作与人联系起来，最终形成了人—工作—交付物（产品）的对应。
  • 项目的组织结构规定了团队拥有的角色和角色之间的关系
  • 项目经理的职权间接反映了项目重要性
  • 矩阵式
  • 集成产品开发团队（integrated Product Team, IPT）
  • 敏捷团队

矩阵式组织

IPT

  • 为配合集成产品开发而形成的一个多学科、跨部门团队，用于复杂产品的开发。
  • IPT将所有的项目相关方集中在一起进行协同开发。
  • IPT可以在项目开发各个层级上形成一个“分形”结构。

敏捷团队

  • 适合项目敏捷开发
  • 快速应变
  • 小规模
  • 高自主性
  • 成员多能

组织结构设计的考虑因素

  • 战略

  • 规模

  • 技术

  • 环境不确定性

项目治理

项目相关方作出决策的流程和框架

  • 定义项目角色和职责
  • 追责机制
  • 评价项目经理的有效性

项目治理原则

  • 原则一：为项目成功确定唯一的终责点
  • 原则二：服务交付主权决定项目主权
  • 原则三：确保干系人管理与项目决策活动相分离
  • 原则四：确保项目治理与组织治理相分离

原则1-为项目成功确定唯一的终责点：个人角色，终责

  • 在项目治理层面上，每个人的角色都必须是非常清晰的。
  • 项目中最重要的终责就是对项目成功的终责。
    • 项目治理中，必须有一个人对项目成败负责，否则没有人领导项目，也就没有人会真正推动项目前行。
  • 此外，还必须由正确的人来承担对项目成功的终责。
    • 一方面，承担终责的人必须在组织内拥有足够的职权，以确保他们有权为项目成功做出必要的决策。
    • 另一方面，这个人必须来自组织中正确的部门或其他领域。

原则2-服务交付主权决定项目主权：主权人、终责人，服务和资产

  • 要首先识别 使用或运营项目 结果的相关方，明确服务交付的主权人（owner），然

后据此确定项目交付的终责人。

  • 项目是组织达到新服务水平的手段，项目交付主要是服务而不仅仅是资产，资产是

服务交付的使能器。

原则3-确保干系人管理与项目决策活动相分离：相关方和治理机构

  • 项目治理机构的主要作用是制定决策，要与相关方管理分开。
    • 好处是只有那些对项目成功绝对重要的相关方，才能参加决策机构，提高决策效率。
    • 带来的问题是，未能参加决策机构的相关方的利益可能受到损失，引发项目矛盾，反而降低决策效率。

原则4-确保项目治理与组织治理相分离：决策机构（组织治理结构）和项目治理结构

  • 正是因为组织治理结构无法为成功交付项目提供必要的框架，才需要建立项目治理

结构。

  • 组织治理：董事会（决策）、经理层（执行）、监事会（监察）
  • 遵守这一原则，就可以尽量减少项目决策层的数量，克服与之有关的时间延误及效

率低下，它将确保项目决策机构有权及时地做出决策

权力

定义和分类：正式权力和个人权力

  • 权力指个体或群体影响其他个体和群体的行为或信仰的能力。
    • 正式权力：个体在组织中的职位为基础
    • 法定权力：根据在组织中的职位支配组织资源的正式职权
    • 强制权力：来自对不服从命令导致的惩罚的惧怕
    • 奖赏权力：来自对服从命令获得奖励的渴望
  • 个人权力：
    • 专家权力：来自专业知识和技能
    • 参照权力：来自他人的尊重和敬佩

指挥权

  • 各部门（角色）职权构成组织的指挥链，应满足
    • 统一指挥原则
    • 连续分级原则

授权：定义，原则，职权分裂

  • 管理者将自己的部分决策权或工作负担转授给下属的过程：
    • 第一，将任务委派给接受授权的下属，并明确应取得的成果；
    • 第二，将完成任务所必需的职权授予下属；
    • 第三，使下属承担起相应的义务。
  • 授权的绝对性原则
    • 责任不可下授
  • 职权分裂：做决策需要汇总多个管理者职权
    • 往往需要联合协调机制

团队组建

  • 为角色选拔任用合适的人选
    • 路线确定之后，干部就是决定的因素
    • 用人不疑，疑人不用

个体作为一个投入产出系统

个体是一个自主性资源

  • 自由意识
  • 有限理性

影响个体实际产出结果的主要因素

能力：个体成功完成工作中各项任务的可能性

  • 智力
  • 体质
  • 情绪
  • 动机：个体为实现目标而付出努力的强度、方向和持续性。
  • “运气”：由环境赋予的、不受个体控制的影响产出绩效的不确定性
    • 或多或少总是存在
    • 扭曲“过程”对“结果”的真实影响
    • 影响人们按“结果”进行激励的公平感

高效团队特征

情景/构成/过程

成员的能力

  • 具有专业的技术
  • 具有解决问题的能力
  • 具有良好的人际技能

成员性格

  • 责任心和经验开放程度

大五人格模型

大五人格模型（big five model）：有五种基本人格维度组成多种人格类型。

  • 外倾性：个体善于交际、健谈、果断以及善于与他人相处的程度。
  • 随和性：个体脾气温和、善于合作和值得信任的程度。
  • 尽责性：个体值得信赖、负责任、可靠、执着和以成就为导向的程度。
  • 情绪稳定性：个体保持冷静、热情、安全感（积极的）或者紧张、忧虑、沮丧和缺乏安全感（消极的）程度。
  • 开放性：个体聪明、涉猎广泛、富有想象力、创造性和好奇心以及具有艺术敏感性的程度。

大量研究表明，尽责性可以有效预测工作绩效
