基于关键路径法的项目管理

关键路径法（Critical Path Method, CPM）是一个进度规划方法。以CPM为基础，发展起一套项目管理体系

什么是CPM

关键路径技术

用网络图的形式来表示项目中工作和工作之间的先后关系，并通过计算工作时间参数，识别项目关键（最长耗时）工作路径

关键路径法（Critical Path Method，CPM）

用节点表示项目工作，用弧表示工作之间先后关系

计划评审技术（Program Evaluation and Review Technique，PERT）

用弧表示工作，用节点表示工作的开始或结束

关键路径法

七个工作时间参数

  • 最早开始日期（Early Start Date，ES） ：在满足先后关系、资源限制等制约因素

的情况下，项目工作可以最早开始的时点。

  • 最晚完成日期（Late Finish Date，LF） ：在满足先后关系、资源限制等制约因素

的情况下，项目工作在保证不延误项目完成日期时最晚必须结束的时点。

  • 持续时间（Duration，DU）：完成某项目工作所需的工作时段总数（不包括节假

日或其他非工作时段）。

  • 最早完成日期（Early Finish Date，EF）：等于工作最早开始日期加上工作持续时

间。

  • 最晚开始日期（Late Start Date，LS）：等于工作最晚结束日期减去工作持续时

间。

  • 总浮动时间（Total Float，TF）：在不延误项目完成日期或违反进度制约因素的前

提下，某工作可以推迟的总时间量（从其最早开始日期起算），也称之为总时差。

  • 自由浮动时间（Free Float，FF）：在不延误其紧后工作最早开始日期的前提下，

某工作可以推迟的时间量，也称之为自由时差

工作的先后关系

项目工作之间存在由于技术、管理和环境等原因导致的时间上的依赖关系，称之为项目工作间的先后关系。

  • 紧前活动 (Predecessor Activity) VS. 紧后活动(Successor Activity)
  • 它们不表示输入输出关系

有四种先后关系

  • 完成到开始（Finish to Start，FS）
  • 开始到开始（Start to Start，SS）
  • 完成到完成（Finish to Finish，FF)
  • 开始到完成（Start to Finish，SF)

识别关键路径

关键路径是项目工作时间网络中，从开始到结束的历时最长的工作路径。

  • 它决定了项目总历时，因此是项目进度控制的焦点。

识别步骤

  • 正确绘制网络图；
  • 将所有非FS关系转化为FS关系；
  • 从左往右，计算活动的最早时间；
  • 从右往左，计算活动的最迟时间；
  • 计算总时差和自由时差；
  • 确定关键路径。

正确绘制网络图

  • 网络图只能有唯一的起点和唯一的终点，当多个活动可以同时开始或结束时，需要

引入虚拟的开始或结束活动，虚拟活动的持续时间为零。

  • 网络图从左到右展开，每个活动都具有唯一的标识。
  • 箭尾连接的是紧前活动，箭头指向的是紧后活动，不允许出现箭头循环的回路

将所有非FS关系转化为FS关系

正推：从左往右，计算最早时间

逆推：从右往左，计算最晚时间

计算总时差和自由时差

确定关键路径

CPM项目规划

工作划分

  • 所有项目任务（工作范围）可以用工作分解结构（work breakdown structure，

WBS）详细描述

  • WBS是一个树性层次结构，最低层称为工作包（work package）。
  • 工作包是项目管理的直接对象，是一项具体工作的分解单元。
    • 工作必须是确定的
    • 完成是可以衡量的
    • 工作受进度约束的
    • 责任是明确的

分解思路

工作时间估计

  • 时间估计与可使用的资源（数量和质量）有直接关系。
  • 这意味着项目进度安排与项目预算密不可分。
  • 是进度优先还是费用优先取决于项目目标。
  • 项目估计的原则
    • 初始估计仅考虑“正常”情况（不考虑异常、特殊情况）
    • 最终估计必须考虑异常情况
    • “准确的”估计事先无从判别，推动项目进展的是相关方认可的“好”的估计

好的估计

  • 假设合理

  • 方法规范

  • 逻辑正确

  • 分析详尽

  • 形式美观

估计

估计的两种基本思路

估计的方法

  • 专家判断
  • 类比估算
  • 参数模型法
  • \(\square T, C \square=f\left(P_1, P_2, \ldots\right)\)
  • 三值估计：\(T=(a+4 m+b) / 6\)

工作先后关系

工作先后关系是一种工作时间上的接口

  • 技术上的——产品分解结构（Product Breakdown Structure，PBS）
  • 法律上的
  • 合同要求的
  • 基于行业良好实践的

优化

初始网络图仅仅是制订CPM计划的脚手架，需要在此基础上考虑各种现实约束。

  • 资源的可用日历
  • 各种外部审批节点
  • 不确定性影响
  • ......

发布进度计划

发布进度计划：计划评审

计划评审

  • 获得一致性和相关方承诺

得到批准后，从正式渠道发布给相关方，成为正式的控制基线和行动指南

CPM项目组织

项目团队能力需求

组建团队

  • 矩阵式：从各职能部门选择或由职能部门指派相应人员

角色设置和分配

  • 负责人

  • 执行人

  • 协助者

  • 应知情人

WBS中每一项工作都需要分配给唯一的负责人

  • 通过对人的激励达到对工作的控制

可用RAM（责任分配矩阵）描述工作分配

CPM项目激励

  • 双向激励
    • 完成目标，奖励
    • 未完成目标，惩罚

CPM项目控制

度量

  • 进度度量→进度目标完成情况
    • 里程碑
    • 按期完工率

评价标准、绩效评价

  • 评价标准
    • 允许的进度偏差范围
  • 绩效评价
    • 与预定进度计划比较

控制策略：项目经理主导

集中式控制策略

  • 项目经理主导

监测

  • 监测对象：结果绩效和过程绩效
    • 结果绩效：工作完成情况付物的质量
    • 过程绩效：过程符合预定要求的程度
  • 监测时机
    • 进度计划中规定的控制时间点

进度展示

偏差和引起偏差的原因

  • 普通原因(common cause):系统固有的，由于系统最初创建时的缺陷而产生的。
  • 特殊原因(special cause): 由于特殊的环境，如特殊员工、部分机器和当地环境等。
  • 错误区分偏差原因将无法有效控制偏差
